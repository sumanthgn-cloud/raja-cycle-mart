Lessons Learned (The Hard Parts)

Not everything went smoothly. Here's what I learned the hard way.

Challenge #1: OTP Delivery Failures

Problem: EmailJS free tier has 200 emails/month limit. We ran out mid-month during a contest spike.

Solution: Added a fallback ‚Äî if email fails, display OTP on screen instead.

otp-fallback.js Fallback Logic 
if (emailSendFailed) {
  // Show OTP on screen as backup
  alert(`Email failed. Your OTP is: ${otp}`);
  // User can still verify and enter
}
Common Mistake

Don't rely on a single service without a backup plan. Always have a fallback for critical features like OTP verification.

Challenge #2: Firebase Security Rules

Problem: Firebase defaults to open access. Anyone could delete database entries!

Solution: Implemented proper Firestore security rules:

firestore.rules Security Rules 
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /lucky_entries/{entry} {
      allow read: if true;  // Anyone can read
      allow write: if request.auth != null;  // Only authenticated users can write
    }
  }
}
Remember

Firebase is powerful but insecure by default. Always set up proper security rules before going live. Test them thoroughly.

Challenge #3: Mobile Performance

Problem: Large images caused slow load times on 4G connections.

Solution:

  • Lazy loading: <img loading="lazy">
  • WebP format (50% smaller than JPEG)
  • Removed Font Awesome (switched to inline SVG icons)
  • Result: Lighthouse score 95+ on mobile
