The Challenge: Digitize or Die

Raja Cycle Mart has been serving Tumkur's cycling community since 1987. But in 2025, they faced an existential crisis: customers were moving online.

Amazon was selling cycles cheaper. Competitors had WhatsApp booking. And the shop's only digital presence was a phone number on Google Maps. They needed a way to capture the online market without adding complex overhead.

❌ The Problems

  • Manual phone-only booking
  • No simplified way to retain customers
  • Zero passive revenue
✅ The Solutions
  • Affiliate Marketplace (Passive Income)
  • AI Chatbot (24/7 Booking)
  • Lucky Draw System (Retention)

Feature Highlight: The "Hard" Part

The client wanted a Lucky Draw System to keep customers coming back. The catch? It had to be secure (no fake entries) but cost nothing to run.

Secure OTP-verified entry system

The Solution: Serverless Verification

I used Vercel Serverless Functions to handle the sensitive OTP logic. This ensures the client-side never sees the secret keys, but we don't need a running server.

/api/send-email.js JavaScript (Serverless) 
export default async function handler(req, res) {
  // 1. Generate secure 6-digit OTP
  const otp = Math.floor(100000 + Math.random() * 900000);
  
  // 2. Send via EmailJS (Server-side)
  const emailParams = {
    to_email: req.body.email,
    otp_code: otp,
    user_name: req.body.name
  };
  
  await sendEmail(emailParams);
  
  // 3. Return success (never return the OTP!)
  res.status(200).json({ message: "OTP sent" });
}
