Feature #1: The "Lucky Draw" Gamification Engine

The Problem

Customer retention was terrible. People came once, never returned.

The Solution

A monthly lucky draw contest where:

  • Customers who make purchases get a "lucky code" on their receipt
  • They enter online via OTP-verified form
  • 3 winners announced monthly (via live announcement system)
  • Prizes: Free service vouchers, accessories

OTP Verification System:

Unique Token Generation:

token-generator.js JavaScript 
function generateToken() {
  const date = new Date().toISOString().split('T')[0].replace(/-/g, '');
  const random = Math.random().toString(36).substr(2, 4).toUpperCase();
  return `RC-${date}-${random}`;
  // Output: RC-20250120-A7F3
}

    Lessons Learned (The Hard Parts)

    Challenge #1: OTP Delivery Failures

    Problem: EmailJS free tier has 200 emails/month limit. Ran out mid-month.

    Solution: Added fallback - if email fails, display OTP on screen itself.

    otp-fallback.js JavaScript 
    if (emailSendFailed) {
  alert(`Email failed. Your OTP is: ${otp}`);
}

    Challenge #2: Firebase Security Rules

    Problem: Anyone could delete database entries (Firebase default is open).

    Solution: Implemented proper Firestore rules:

    firestore.rules Firebase Security Rules 
    rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /lucky_entries/{entry} {
      allow read: if true;
      allow write: if request.auth != null; // Only authenticated users
    }
  }
}

    Challenge #3: Mobile Performance

    Problem: Large images caused slow load times on 4G.

    Solution:

    • Lazy loading: <img loading="lazy">
    • WebP format (50% smaller than JPEG)
    • Removed Font Awesome (switched to inline SVG icons)
